# Phase 7.4 — Motion & Animation: Cơ chế thật từ Browser đến Vestibular System

> **Độ ưu tiên:** 🔴 PHẢI BIẾT (`prefers-reduced-motion`, không dùng animation làm thông tin duy nhất) + 🟡 NÊN BIẾT (WCAG 2.3.3 Animation from Interactions)

---

## 1. Cơ chế thật — Tại sao animation có thể gây hại

### Vestibular disorder — Vấn đề y tế thật sự

Vestibular system là hệ thống cân bằng trong tai trong. Khoảng **35% người trưởng thành trên 40 tuổi** có vestibular dysfunction ở mức độ nào đó. Với những người này, animation trên screen có thể trigger:

```
Animation trên screen
        │
        │  Mắt nhìn thấy chuyển động
        ▼
Visual cortex xử lý motion signal
        │
        │  Brain so sánh với vestibular signal (tai trong)
        ▼
CONFLICT: Mắt thấy chuyển động, tai trong báo đứng yên
        │
        ▼
Triệu chứng:
  - Chóng mặt (dizziness)
  - Buồn nôn (nausea)
  - Đau đầu (headaches)
  - Disorientation
  - Trong trường hợp nặng: migraine, vomiting
```

**Đây không phải "khó chịu nhẹ"** — với vestibular disorder nặng, một trang web có parallax scrolling hoặc full-screen transition animation có thể khiến user bệnh cả ngày.

Ngoài vestibular disorder, animation còn ảnh hưởng tới:

- Người có **epilepsy** (động kinh) — flashing content có thể trigger seizure
- Người có **ADHD** — animation distract, giảm khả năng tập trung
- Người có **cognitive disabilities** — animation gây confusion về trạng thái UI

### Browser implement `prefers-reduced-motion` như thế nào

```
OS Level:
  macOS:   System Preferences → Accessibility → Display → Reduce Motion
  Windows: Settings → Ease of Access → Display → Show animations in Windows
  iOS:     Settings → Accessibility → Motion → Reduce Motion
  Android: Settings → Accessibility → Remove animations

        │  OS expose preference qua system API
        ▼

Browser Level:
  Browser query OS preference
  Expose qua CSS media query: prefers-reduced-motion
  Expose qua JS: window.matchMedia('(prefers-reduced-motion: reduce)')

        │
        ▼

Values:
  no-preference → User không set preference (không phải "wants motion"!)
  reduce        → User đã enable "reduce motion" ở OS level
```

**Quan trọng:** `no-preference` nghĩa là **không biết** preference của user, không phải user muốn animation. Default safe assumption là nhiều users không biết setting này tồn tại.

## 2. CSS `prefers-reduced-motion` — Implementation đúng

### Pattern sai vs đúng

```css
/* ❌ Pattern sai — thêm animation là default, opt-out là ngoại lệ */
.card {
  transition:
    transform 0.3s ease,
    box-shadow 0.3s ease;
}
.card:hover {
  transform: translateY(-4px);
  box-shadow: 0 8px 24px rgba(0, 0, 0, 0.15);
}

@media (prefers-reduced-motion: reduce) {
  .card {
    transition: none; /* Cắt hẳn — abrupt, không có alternative */
  }
}
```

```css
/* ✅ Pattern đúng — motion-safe approach */
/* Base styles: không có animation */
.card {
  /* Visual styles không liên quan animation */
  border-radius: 8px;
  background: white;
}

/* Animation chỉ apply khi user OK với motion */
@media (prefers-reduced-motion: no-preference) {
  .card {
    transition:
      transform 0.3s ease,
      box-shadow 0.3s ease;
  }
  .card:hover {
    transform: translateY(-4px);
    box-shadow: 0 8px 24px rgba(0, 0, 0, 0.15);
  }
}

/* Với reduced motion: vẫn có visual feedback, nhưng không có motion */
@media (prefers-reduced-motion: reduce) {
  .card:hover {
    /* Thay animation bằng instant visual change */
    box-shadow: 0 8px 24px rgba(0, 0, 0, 0.15); /* Không transition */
    outline: 2px solid #005fcc; /* Alternative visual indicator */
  }
}
```

### Phân loại animation theo mức độ nguy hiểm

```
┌──────────────────────────────────────────────────────────────┐
│              Animation Risk Classification                    │
├─────────────────────┬────────────────────────────────────────┤
│ HIGH RISK           │ Vestibular trigger — phải disable       │
│ (phải disable)      │                                        │
│                     │ - Parallax scrolling                   │
│                     │ - Large element movement (slide-in)    │
│                     │ - Spinning/rotating elements           │
│                     │ - Zoom animations (scale in/out)       │
│                     │ - Background video autoplay            │
│                     │ - Scroll-triggered animations          │
├─────────────────────┼────────────────────────────────────────┤
│ MEDIUM RISK         │ Có thể gây khó chịu — nên reduce       │
│ (nên reduce)        │                                        │
│                     │ - Page transitions (fade OK, slide NO) │
│                     │ - Expanding panels (height change)     │
│                     │ - Toast notifications flying in        │
│                     │ - Loading skeleton animation           │
├─────────────────────┼────────────────────────────────────────┤
│ LOW RISK            │ Thường OK ngay cả với reduced-motion   │
│ (generally OK)      │                                        │
│                     │ - Opacity fade (không có movement)     │
│                     │ - Color transition                     │
│                     │ - Focus indicator animation            │
│                     │ - Underline grow (text decoration)     │
│                     │ - Very subtle, small-scale movement    │
└─────────────────────┴────────────────────────────────────────┘
```

```css
/* ✅ Fade transition — safe ngay cả với reduced-motion users */

  opacity: 1;
}

  transform: translateY(0);

  }
```

## 3. JavaScript và `prefers-reduced-motion`

### Detect và respond trong JS

```javascript
// Query preference
const prefersReducedMotion = window.matchMedia(
  '(prefers-reduced-motion: reduce)',
);

// Check current value
console.log(prefersReducedMotion.matches); // true nếu user muốn reduced motion

// Listen for changes — user có thể thay đổi setting trong khi app đang chạy
prefersReducedMotion.addEventListener('change', (event) => {
  if (event.matches) {
    // User bật reduce motion — disable animations ngay lập tức
    disableAnimations();
  } else {
    // User tắt reduce motion
    enableAnimations();
  }
});

// Utility function
function shouldAnimate() {
  return !window.matchMedia('(prefers-reduced-motion: reduce)').matches;
}
```

### Web Animations API với reduced-motion

```javascript
// ❌ Ignore preference — luôn animate
function openMenu(menuEl) {
  menuEl.animate(
    [
      { transform: 'translateX(-100%)', opacity: 0 },
      { transform: 'translateX(0)', opacity: 1 },
    ],
    { duration: 300, easing: 'ease-out', fill: 'forwards' },
  );
}

// ✅ Respect preference
function openMenu(menuEl) {
  const prefersReduced = window.matchMedia(
    '(prefers-reduced-motion: reduce)',
  ).matches;

  if (prefersReduced) {
    // Instant show — không có spatial movement
    menuEl.style.opacity = '1';
    menuEl.style.transform = 'none';
    // Hoặc: animate chỉ opacity nếu muốn có transition
    menuEl.animate([{ opacity: 0 }, { opacity: 1 }], {
      duration: 150,
      fill: 'forwards',
    });
  } else {
    // Full animation cho users OK với motion
    menuEl.animate(
      [
        { transform: 'translateX(-100%)', opacity: 0 },
        { transform: 'translateX(0)', opacity: 1 },
      ],
      { duration: 300, easing: 'ease-out', fill: 'forwards' },
    );
  }

  menuEl.hidden = false;
}
```

### React pattern — Animation với reduced-motion

```jsx
import { useEffect, useState } from 'react';

// Custom hook để track preference
function usePrefersReducedMotion() {
  const [prefersReducedMotion, setPrefersReducedMotion] = useState(() => {
    // Initial value từ media query
    return window.matchMedia('(prefers-reduced-motion: reduce)').matches;
  });

  useEffect(() => {
    const mediaQuery = window.matchMedia('(prefers-reduced-motion: reduce)');

    const handleChange = (event) => {
      setPrefersReducedMotion(event.matches);
    };

    mediaQuery.addEventListener('change', handleChange);
    return () => mediaQuery.removeEventListener('change', handleChange);
  }, []);

  return prefersReducedMotion;
}

// Component dùng hook
function AnimatedCard({ children }) {
  const prefersReducedMotion = usePrefersReducedMotion();

  return (
    <div
      style={{
        // Conditional animation properties
        transition: prefersReducedMotion
          ? 'none' // No animation
          : 'transform 0.3s ease, box-shadow 0.3s ease',
      }}
      onMouseEnter={(e) => {
        if (!prefersReducedMotion) {
          e.currentTarget.style.transform = 'translateY(-4px)';
        }
        // Visual feedback vẫn có, chỉ không có motion
        e.currentTarget.style.boxShadow = '0 8px 24px rgba(0,0,0,0.15)';
      }}
      onMouseLeave={(e) => {
        e.currentTarget.style.transform = '';
        e.currentTarget.style.boxShadow = '';
      }}
    >
      {children}
    </div>
  );
}

// Framer Motion integration — library phổ biến nhất
import { motion, useReducedMotion } from 'framer-motion';

function PageTransition({ children }) {
  const shouldReduceMotion = useReducedMotion();

  const variants = {
    hidden: {
      opacity: 0,
      // Chỉ có slide khi user OK với motion
      y: shouldReduceMotion ? 0 : 20,
    },
    visible: {
      opacity: 1,
      y: 0,
    },
  };

  return (
    <motion.div
      initial='hidden'
      animate='visible'
      variants={variants}
      transition={{
        duration: shouldReduceMotion ? 0.1 : 0.4,
        ease: 'easeOut',
      }}
    >
      {children}
    </motion.div>
  );
}
```

## 4. Flashing Content — WCAG 2.3.1 Three Flashes

### Tại sao flash content nguy hiểm

Photosensitive epilepsy ảnh hưởng khoảng 1/4000 người. Content flash với tần số trong range 3-50Hz có thể trigger seizure.

**WCAG 2.3.1 Three Flashes or Below Threshold (Level A — bắt buộc):**

```
FAIL nếu content:
  - Flash hơn 3 lần trong 1 giây
  - VÀ vượt ngưỡng về size và contrast

Safe nếu:
  - Flash ≤ 3 lần/giây
  - HOẶC flash area nhỏ hơn threshold (< 10 degrees of visual field)
  - HOẶC contrast của flash thấp
  - HOẶC red component của flash thấp (red flash đặc biệt nguy hiểm)
```

```javascript
// ❌ NGUY HIỂM — có thể trigger seizure
function dangerousFlash() {
  let count = 0;
  const interval = setInterval(() => {
    document.body.style.background = count % 2 === 0 ? '#ff0000' : '#ffffff';
    count++;
    if (count > 20) clearInterval(interval); // 20 flashes trong ~1s = ~10Hz
  }, 50);
}

// ✅ Safe alert animation — không flash
function safeAlert(element) {
  // Một lần fade in — không repeat, không flash
  element.animate([{ opacity: 0 }, { opacity: 1 }], {
    duration: 300,
    fill: 'forwards',
  });
}

// ✅ Nếu cần "attention" animation — slow, below 3Hz
function gentlePulse(element) {
  // 1 pulse per 600ms = ~1.67Hz — safe
  element.animate([{ opacity: 1 }, { opacity: 0.7 }, { opacity: 1 }], {
    duration: 1200, // > 333ms per flash = < 3Hz
    iterations: 3, // Giới hạn số lần
  });
}
```

## 5. Animation không được là thông tin duy nhất

### WCAG 1.3.3 — Sensory Characteristics

Thông tin không được chỉ dựa vào một sensory characteristic duy nhất. Animation nằm trong category này.

```javascript
// ❌ Chỉ dùng animation để convey state — broken nếu animation disabled
function showFormSuccess() {
  // Chỉ có animation — nếu animation disabled, không có feedback nào
  submitButton.classList.add('success-bounce');

  setTimeout(() => {
    submitButton.classList.remove('success-bounce');
  }, 1000);
}

// ✅ Animation + text + aria announcement
function showFormSuccess() {
  // 1. Visual text — visible ngay cả khi animation off
  const statusEl = document.getElementById('form-status');
  statusEl.textContent = '✓ Changes saved successfully';
  statusEl.className = 'status status--success';

  // 2. Animation — enhanced experience, không phải sole indicator
  if (!window.matchMedia('(prefers-reduced-motion: reduce)').matches) {
    submitButton.classList.add('success-bounce');
    setTimeout(() => {
      submitButton.classList.remove('success-bounce');
    }, 600);
  }

  // 3. Screen reader announcement
  // statusEl có role="status" (aria-live="polite") → tự announce
}
```

## 6. WCAG 2.3.3 — Animation from Interactions (AAA)

### Spec và practical application

WCAG 2.3.3 là Level AAA — aspirational, không bắt buộc. Nhưng tinh thần của nó quan trọng:

> **Motion animation triggered by interaction can be disabled, unless the animation is essential to the functionality or the information being conveyed.**

```
"Essential" animation examples:
  - Progress bar — vị trí của bar IS the information
  - Video playback — motion IS the content
  - Map panning — movement IS the interaction

"Non-essential" animation examples:
  - Page slide transitions
  - Parallax scrolling effects
  - Decorative hover animations
  - Celebratory confetti on completion
```

### User control over animation — Beyond OS setting

Tốt hơn chỉ rely vào OS setting: cho user control trực tiếp trong app.

```jsx
// Animation preference toggle trong app
function AnimationPreference() {
  const [motionEnabled, setMotionEnabled] = useState(() => {
    // Check saved preference hoặc OS default
    const saved = localStorage.getItem('motion-preference');
    if (saved) return saved === 'enabled';
    return !window.matchMedia('(prefers-reduced-motion: reduce)').matches;
  });

  const toggle = () => {
    const newValue = !motionEnabled;
    setMotionEnabled(newValue);
    localStorage.setItem(
      'motion-preference',
      newValue ? 'enabled' : 'disabled',
    );

    // Apply globally
    document.documentElement.setAttribute(
      'data-motion',
      newValue ? 'enabled' : 'reduced',
    );
  };

  return (
    <div>
      <label>
        <input
          type='checkbox'
          checked={motionEnabled}
          onChange={toggle}
          aria-describedby='motion-desc'
        />
        Enable animations
      </label>
      <p
        id='motion-desc'
        className='hint'
      >
        Turn off if animations cause discomfort
      </p>
    </div>
  );
}
```

```css
/* CSS respond to data attribute từ user preference */
/* Kết hợp OS preference VÀ app-level preference */

/* Default: respect OS */
@media (prefers-reduced-motion: no-preference) {
  .animated-element {
    transition: transform 0.3s ease;
  }
}

/* OS says reduce — override regardless of app setting */
@media (prefers-reduced-motion: reduce) {
  .animated-element {
    transition: none !important;
    animation: none !important;
  }
}

/* App-level: user explicitly disabled trong app */
[data-motion='reduced'] .animated-element {
  transition: none !important;
  animation: none !important;
}
```

```javascript
class ToastSystem {
  constructor() {
    this.container = document.getElementById('toast-container');
    this.toasts = new Map(); // id → element
  }

  show({ message, type = 'info', duration = 5000, id = crypto.randomUUID() }) {
    const toast = this.createToast({ message, type, id });

    // Inject vào aria-live region → tự announce vì container có aria-live="polite"
    this.container.appendChild(toast);
    this.toasts.set(id, toast);

    // Auto dismiss
    if (duration > 0) {
      setTimeout(() => this.dismiss(id), duration);
    }

    return id; // Return id để caller có thể manually dismiss
  }

  createToast({ message, type, id }) {
    const toast = document.createElement('div');
    toast.className = `toast toast--${type}`;
    toast.dataset.toastId = id;

    // role="status" cho non-critical, role="alert" cho errors
    toast.setAttribute('role', type === 'error' ? 'alert' : 'status');

    // Icon — aria-hidden vì type class convey visual meaning
    // Text convey semantic meaning
    const icons = { success: '✓', error: '✕', info: 'ℹ' };

    toast.innerHTML = `
      <span aria-hidden="true" class="toast__icon">${icons[type]}</span>
      <span class="toast__message">${this.escapeHTML(message)}</span>
      <button 
        class="toast__close"
        aria-label="Dismiss notification"
        type="button"
      >
        <span aria-hidden="true">×</span>
      </button>
    `;

    toast.querySelector('.toast__close').addEventListener('click', () => {
      this.dismiss(id);
    });

    return toast;
  }

  dismiss(id) {
    const toast = this.toasts.get(id);
    if (!toast) return;

    const prefersReduced = window.matchMedia(
      '(prefers-reduced-motion: reduce)',
    ).matches;

    if (prefersReduced) {
      // Instant remove — không có leave animation
      toast.remove();
      this.toasts.delete(id);
    } else {
      // Animate out rồi remove
      toast.classList.add('is-leaving');
      toast.addEventListener(
        'animationend',
        () => {
          toast.remove();
          this.toasts.delete(id);
        },
        { once: true },
      );
    }
  }

  escapeHTML(str) {
    const div = document.createElement('div');
    div.appendChild(document.createTextNode(str));
    return div.innerHTML;
  }
}

// Usage
const toasts = new ToastSystem();

// Success
toasts.show({ message: 'Profile updated successfully', type: 'success' });

// Error — role="alert" → assertive announcement
toasts.show({
  message: 'Failed to save. Please try again.',
  type: 'error',
  duration: 0, // Không auto-dismiss — user phải dismiss manually
});

// Info với custom duration
toasts.show({
  message: 'Your session will expire in 5 minutes',
  type: 'info',
  duration: 8000,
});
```

### Next.js page transitions

```jsx
// ❌ Page transition phổ biến — không respect reduced-motion
import { AnimatePresence, motion } from 'framer-motion';

function MyApp({ Component, pageProps, router }) {
  return (
    <AnimatePresence mode='wait'>
      <motion.div
        key={router.route}
        initial={{ opacity: 0, x: -200 }} // Slide từ trái — HIGH RISK
        animate={{ opacity: 1, x: 0 }}
        exit={{ opacity: 0, x: 200 }}
        transition={{ duration: 0.3 }}
      >
        <Component {...pageProps} />
      </motion.div>
    </AnimatePresence>
  );
}

// ✅ Respect reduced-motion
import { useReducedMotion } from 'framer-motion';

function PageTransition({ children, routeKey }) {
  const shouldReduce = useReducedMotion();

  const variants = shouldReduce
    ? {
        // Reduced: fade only, no movement
        initial: { opacity: 0 },
        animate: { opacity: 1 },
        exit: { opacity: 0 },
      }
    : {
        // Full: fade + slide
        initial: { opacity: 0, x: -20 },
        animate: { opacity: 1, x: 0 },
        exit: { opacity: 0, x: 20 },
      };

  return (
    <AnimatePresence mode='wait'>
      <motion.div
        key={routeKey}
        variants={variants}
        initial='initial'
        animate='animate'
        exit='exit'
        transition={{ duration: shouldReduce ? 0.15 : 0.3 }}
      >
        {children}
      </motion.div>
    </AnimatePresence>
  );
}
```

## 10. Câu hỏi kiểm tra

**Q1: Tại sao `prefers-reduced-motion: no-preference` không có nghĩa là "user muốn animation"?**

> `no-preference` là giá trị default khi user chưa set bất kỳ preference nào trong OS settings. Hầu hết users không biết setting "Reduce Motion" tồn tại, không phải họ đã xem xét và quyết định muốn animation. Một số lượng significant trong số `no-preference` users thực ra có vestibular issues hoặc photosensitivity nhưng chưa tìm được OS setting. Nên design default là "minimal motion" và add motion như enhancement, không phải design với full motion và remove khi `reduce`. Pattern đúng: wrap animation trong `@media (prefers-reduced-motion: no-preference)` thay vì override trong `reduce`.

**Q2: Điều gì xảy ra nếu toast notification inject content vào aria-live region trong khi animation đang chạy?**

> Announcement có thể bị delay hoặc miss. Screen reader detect mutation khi content thay đổi trong live region — nếu element mới được append nhưng bắt đầu là `opacity: 0` và animate tới `opacity: 1`, content đã ở trong DOM từ đầu. Điều này thường ổn vì screen reader đọc text content, không care về opacity. Nhưng nếu animation involve thay đổi text content trong quá trình animate (type-writer effect), screen reader có thể announce các intermediate states. Fix: set final content ngay, chỉ animate visual appearance.

**Q3: Đoạn code này có vấn đề gì?**

```css
@media (prefers-reduced-motion: reduce) {
  * {
    animation: none;
    transition: none;
  }
}
```

> Hai vấn đề. Thứ nhất, `animation: none` và `transition: none` set duration về 0 — một số browser không fire `animationend` event khi duration = 0, gây bugs trong JS code chờ event này để remove classes, cleanup state. Dùng `animation-duration: 0.01ms` thay thế. Thứ hai, cách tiếp cận "opt-out" — design với full animation rồi strip trong reduced-motion — không phải best practice. Pattern tốt hơn là design base styles không có animation, thêm animation trong `@media (prefers-reduced-motion: no-preference)`. Global `* { animation: none }` cũng override animations mà user có thể cần (progress indicators, loading states) mà không cung cấp alternative.

## Câu hỏi ôn tập (NotebookLM)

### Q1: Tại sao "disable toàn bộ animation" không phải giải pháp đúng cho accessibility, và alternative đúng là gì?

**Đáp án:**

"Disable toàn bộ animation" là cách tiếp cận sai vì animation phục vụ nhiều mục đích quan trọng khác nhau, không phải tất cả đều problematic:

**Animation có mục đích functional:**

- Progress indicators communicate state change — không có animation, user không biết action đã được nhận
- Focus indicator animation giúp user track focus position
- Collapse/expand animation giúp user understand spatial relationship (content ở đâu đến đâu)
- Loading skeleton communicate "placeholder, content sắp đến"

**Khi disable hoàn toàn:** Những animations này cũng biến mất → trải nghiệm worse, không better. User với reduced-motion không cần "zero animation" — họ cần "không có motion gây discomfort."

**Alternative đúng là phân loại và replace:**

1. **HIGH RISK animations** (parallax, large-scale movement, zoom, spin): Remove hoàn toàn hoặc replace với instant state change
2. **MEDIUM RISK animations** (slide transitions, expanding panels): Replace với safer alternative — fade (opacity only, không có spatial movement), hoặc instant với visual differentiation
3. **LOW RISK animations** (opacity fade, color transition, subtle scale < 5%): Có thể giữ nguyên, ít/không gây vestibular issues
4. **FUNCTIONAL animations** (progress bar, focus indicator): Phải giữ, có thể reduce duration hoặc intensity

Pattern implementation đúng:

- Base styles không có animation
- Wrap tất cả non-essential, motion-heavy animations trong `@media (prefers-reduced-motion: no-preference)`
- Trong `@media (prefers-reduced-motion: reduce)`: provide safe alternatives, không chỉ strip
- Provide in-app control để user fine-tune preference

### Q2: Giải thích cơ chế tại sao aria-live region phải có animation được implement đúng cách để screen reader announcement hoạt động correctly.

**Đáp án:**

Screen reader detect content changes trong `aria-live` regions thông qua **Mutation Observer** internal của browser — browser watch các DOM mutations (text changes, node insertions/removals) trong subtrees được đánh dấu với `aria-live`.

**Interaction với animation có ba scenarios:**

**Scenario 1 — Animation chỉ thay đổi CSS (opacity, transform):**
Content đã có trong DOM từ đầu, chỉ visual appearance thay đổi. Mutation Observer không fire vì text content và DOM structure không thay đổi. Screen reader đã có thể đọc element khi nó được thêm vào DOM — announcement xảy ra đúng thời điểm. Animation visual không ảnh hưởng.

**Scenario 2 — Content được inject đồng thời với animation:**

```javascript
// Toast: tạo element, set text, append vào live region, animate
const toast = document.createElement('div');
toast.textContent = 'Saved successfully'; // Set content
container.appendChild(toast); // Trigger mutation → trigger announcement
toast.classList.add('animate-in'); // Animate — sau announcement
```

Announcement xảy ra khi append — trước khi animation start. Ổn.

**Scenario 3 — Content được set qua animated typewriter effect:**

```javascript
// Type từng character — mỗi character là một mutation
let i = 0;
const text = 'Item saved';
const interval = setInterval(() => {
  el.textContent += text[i++]; // Mutation mỗi character!
  if (i >= text.length) clearInterval(interval);
}, 50);
```

Screen reader có thể announce "I", "It", "Ite", "Item"... mỗi intermediate state. Hoặc debounce internal của screen reader buffer các announces. Behavior không predict được — avoid pattern này trong live regions.

**Rule:** Live region content phải được set **toàn bộ trong một operation**. Animation visual là fine, nhưng text content trong live region phải đi từ empty → final text trong một DOM mutation, không nhiều mutations nhỏ.

### Q3: Mô tả đầy đủ cách implement một button "Save" có loading state đúng chuẩn accessibility, bao gồm cả animation và motion considerations.

**Đáp án:**

Button "Save" với loading state cần communicate với ba audiences: sighted mouse users, sighted keyboard users, và screen reader users — đồng thời respect motion preferences.

**HTML Structure:**

```html
<button
  id="save-btn"
  type="button"
  aria-busy="false"
>
  <span
    class="btn__icon"
    aria-hidden="true"
    >💾</span
  >
  <span class="btn__text">Save changes</span>
  <span
    class="btn__spinner"
    aria-hidden="true"
    hidden
  ></span>
  <span
    class="sr-only"
    role="status"
    id="save-status"
  ></span>
</button>
```

**States và transitions:**

_Idle state:_ Button enabled. aria-busy="false". Text "Save changes". Icon visible. Spinner hidden.

_Loading state (sau click):_

1. Set `aria-busy="true"` — AT biết action đang process
2. Set `disabled` — prevent double-submit
3. Change button text: "Saving..." — visual text thay đổi
4. Show spinner, hide icon
5. Inject vào `#save-status` (role="status", aria-live="polite"): "Saving your changes, please wait" — screen reader announcement
6. Animation: spinner chỉ animate nếu `prefers-reduced-motion: no-preference`. Nếu reduce: static spinner hoặc opacity pulse (không rotate)

_Success state:_

1. Remove `disabled`, set `aria-busy="false"`
2. Change button text: "Saved ✓"
3. Hide spinner, show checkmark icon
4. Inject vào `#save-status`: "Changes saved successfully"
5. Animation: nếu no-preference, brief scale animation trên button (100% → 103% → 100%) để celebrate. Nếu reduce: instant state change, chỉ text thay đổi.
6. After 2 seconds: reset về idle state

_Error state:_

1. Remove `disabled`, set `aria-busy="false"`
2. Keep focus trên button — user cần retry
3. Change button text: "Save failed — Try again"
4. Inject vào error live region (role="alert", assertive): "Failed to save changes. [error reason]. Please try again." — assertive vì cần immediate attention
5. Animation: nếu no-preference, brief shake animation (horizontal translate) — universally understood error indicator. Nếu reduce: instant, chỉ color change (button background sang red briefly)

**Motion considerations:**

- Spinner rotation: disable khi reduce-motion, replace với opacity pulse
- Success scale animation: disable khi reduce-motion, instant change
- Error shake: disable khi reduce-motion, replace với color flash (brief background-color change — không phải spatial motion)
- Tất cả text changes và ARIA state changes xảy ra independent của animation — không phải "animation xong rồi mới update ARIA"
