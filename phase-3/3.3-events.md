### A — Từ hardware đến JavaScript: Con đường thật của một event

Trước khi đến `addEventListener`, một browser event trải qua nhiều tầng:

```
User nhấn chuột vào button
        ↓
Hardware interrupt → OS kernel
        ↓
OS: tạo mouse event, deliver tới process có focused window
        ↓
Browser process (UI thread): nhận OS event
        ↓
Browser process: identify target window/tab
        ↓
IPC (Inter-Process Communication) → Renderer process
        ↓
Compositor thread: biết scroll offset, layer positions
→ Identify visual element bị hit (hit testing)
        ↓
Main thread: nhận hit-test result
→ Build DOM event object
→ Determine event target (DOM node)
→ Dispatch event qua DOM tree
        ↓
JavaScript event listeners execute
```

**Hit testing xảy ra trên compositor thread**, không phải main thread. Đây là lý do scroll có thể smooth ngay cả khi main thread bận — compositor thread xử lý scroll independently. Chỉ khi cần notify JS (scroll event listeners) thì mới cần main thread.

**Event object được tạo trong Blink (C++)**, sau đó được wrapped và exposed sang JavaScript qua bindings. Mỗi property bạn access trên event object (`event.target`, `event.clientX`) là cross-boundary call.

### B — Event Propagation: 3 phases thật sự

Hầu hết developer biết "bubble" nhưng không biết đủ sâu để debug khi có vấn đề.

#### Spec định nghĩa 3 phases:

```
document
└── html
    └── body
        └── div.container
            └── button  ← user click vào đây
```

```
Phase 1 — CAPTURE (từ root xuống target):
document → html → body → div.container → button

Phase 2 — AT_TARGET (tại target element):
button (cả capture và bubble listeners đều fire ở đây)

Phase 3 — BUBBLE (từ target lên root):
button → div.container → body → html → document → window
```

**Thứ tự chính xác trên target element:**

Đây là phần ít người biết: khi target là chính element bị click, **capture và bubble listeners đều chạy ở phase AT_TARGET**, và thứ tự là **theo thứ tự được registered**, không phải capture trước bubble:

```javascript
const button = document.querySelector('button');

button.addEventListener('click', () => console.log('bubble 1')); // registered first
button.addEventListener('click', () => console.log('capture 1'), true); // registered second
button.addEventListener('click', () => console.log('bubble 2')); // registered third

// Khi click vào button:
// Output: "bubble 1" → "capture 1" → "bubble 2"
// (thứ tự registration, không phải phase)
```

**Nhưng trên ancestor elements**, capture listeners luôn chạy trước bubble:

```javascript
document.addEventListener('click', () => console.log('doc capture'), true);
document.addEventListener('click', () => console.log('doc bubble'));
div.addEventListener('click', () => console.log('div capture'), true);
div.addEventListener('click', () => console.log('div bubble'));
button.addEventListener('click', () => console.log('button'));

// Click button → output:
// "doc capture"    ← capture phase, từ root
// "div capture"    ← capture phase
// "button"         ← AT_TARGET (cả capture + bubble)
// "div bubble"     ← bubble phase
// "doc bubble"     ← bubble phase, về root
```

#### Không phải event nào cũng bubble

```javascript
// Events KHÔNG bubble:
focus; // dùng focusin thay thế (bubble)
blur; // dùng focusout thay thế (bubble)
scroll; // không bubble (trừ document scroll)
mouseenter; // không bubble (dùng mouseover để bubble)
mouseleave; // không bubble (dùng mouseout để bubble)
load; // không bubble
unload; // không bubble
resize; // không bubble

// Events CÓ bubble:
click;
dblclick;
(mousedown, mouseup, mousemove);
(mouseover, mouseout); // bubble, khác mouseenter/mouseleave
(keydown, keyup, keypress);
input;
change;
submit;
```

**mouseenter vs mouseover — sự khác biệt cơ chế:**

```
<div class="parent">     ← mouseover fire khi chuột vào đây
  <span>child</span>     ← mouseover fire lần nữa khi chuột vào span
</div>                      (bubble từ span lên div)

mouseenter: chỉ fire một lần khi chuột vào div
            KHÔNG fire khi chuột move giữa div và span bên trong
mouseover:  fire mỗi khi chuột vào div hoặc bất kỳ descendant nào
            (kể cả từ child bubble lên)
```

### C — addEventListener Options: Sâu hơn true/false

```javascript
element.addEventListener(type, listener, options);
```

`options` là object với các properties:

#### `capture: boolean` (default: false)

```javascript
// Cũ — chỉ có useCapture boolean:
el.addEventListener('click', handler, true); // capture phase
el.addEventListener('click', handler, false); // bubble phase

// Mới — options object:
el.addEventListener('click', handler, { capture: true });
el.addEventListener('click', handler, { capture: false });
```

**Khi nào dùng capture:**

- Intercept event trước khi nó đến target
- Implement modal/overlay: chặn click ra ngoài modal
- Analytics: track tất cả clicks trước khi app handlers chạy

#### `once: boolean` (default: false)

```javascript
el.addEventListener('click', handler, { once: true });
// Handler tự động bị removed sau lần đầu tiên fire
// Tương đương:
el.addEventListener('click', function handler(e) {
  el.removeEventListener('click', handler);
  // ... code
});
```

**Cơ chế:** Browser giữ internal flag. Sau khi listener được invoked, browser gọi `removeEventListener` internally. Không cần lưu reference để remove sau.

**Use case:** Animation "play once", dialog open, first-time tutorials.

#### `passive: boolean` (default: depends)

Đây là option quan trọng nhất cho performance.

**Vấn đề trước `passive`:**

```
User scroll trang
      ↓
Browser muốn scroll ngay lập tức (compositor thread)
      ↓
NHƯNG: Có scroll event listener không?
      ↓
Browser PHẢI chờ main thread execute listener
(vì listener có thể gọi preventDefault() để cancel scroll)
      ↓
Nếu main thread bận → scroll bị delay → jank
```

**Với `passive: true`:**

```javascript
window.addEventListener('scroll', handler, { passive: true });
// Browser biết: listener này sẽ KHÔNG gọi preventDefault()
// → Scroll xảy ra ngay trên compositor thread
// → Không cần chờ main thread
// → Smooth scroll kể cả khi main thread bận
```

**Default values của `passive`:**

Từ Chrome 56+, browser tự set `passive: true` cho `touchstart` và `touchmove` trên `window`, `document`, và `document.body`. Đây là lý do tại sao gọi `preventDefault()` trong touch handlers hay gặp warning:

```
[Intervention] Unable to preventDefault inside passive event listener
due to target being treated as passive.
```

**Khi nào KHÔNG dùng passive:**

```javascript
// ❌ Sẽ không work — passive listener không thể preventDefault
el.addEventListener(
  'touchmove',
  (e) => {
    e.preventDefault(); // Ignored! Warning in console
  },
  { passive: true },
);

// ✅ Cần cancel default behavior → không dùng passive
el.addEventListener(
  'touchmove',
  (e) => {
    e.preventDefault(); // Works — nhưng scroll sẽ bị chặn hoàn toàn
  },
  { passive: false },
);
```

#### `signal: AbortSignal`

```javascript
const controller = new AbortController();

el.addEventListener('click', handler, { signal: controller.signal });
el.addEventListener('keydown', otherHandler, { signal: controller.signal });

// Remove tất cả listeners cùng lúc:
controller.abort();
// Tương đương removeEventListener cho từng cái, nhưng clean hơn nhiều
```

**Use case:** Cleanup listeners khi component unmount, cancel listeners khi navigation.

### D — Event Delegation: Tại sao nó hiệu quả

#### Vấn đề: N listeners cho N elements

```javascript
// ❌ Naive approach
document.querySelectorAll('.product-card').forEach((card) => {
  card.addEventListener('click', handleCardClick);
  // 1000 products → 1000 event listeners
  // Mỗi listener là một object trong memory
  // Khi products load thêm (infinite scroll) → phải manually attach listeners mới
});
```

#### Event Delegation: 1 listener xử lý tất cả

```javascript
// ✅ Event delegation — 1 listener trên container
const productList = document.querySelector('.product-list');

productList.addEventListener('click', (event) => {
  // event.target: element bị click THỰC SỰ (có thể là child của card)
  // event.currentTarget: element có listener (productList)

  // Dùng closest() để tìm ancestor match selector
  const card = event.target.closest('.product-card');
  if (!card) return; // Click vào vùng không phải card

  handleCardClick(card);
});
```

**Tại sao `closest()` thay vì check `event.target` trực tiếp:**

```html
<div class="product-card">
  ← Muốn detect click vào đây
  <img
    src="..."
    alt="..."
  />
  ← Nhưng user có thể click vào img
  <div class="info">
    <h3>Product Name</h3>
    ← Hoặc click vào h3
    <span class="price">$99</span>
  </div>
</div>
```

```javascript
// ❌ Chỉ check target — miss khi click vào children
if (event.target.classList.contains('product-card')) {
  // Chỉ work khi click đúng vào div.product-card
  // Miss khi click vào img, h3, span bên trong
}

// ✅ closest() — traverse lên tìm ancestor match
const card = event.target.closest('.product-card');
// Dù click vào img, h3, span → closest() tìm được card ancestor
// Nếu click ra ngoài card → trả về null
```

**`closest()` cơ chế:** Traverse từ element hiện tại lên các ancestors, trả về element đầu tiên match selector. Dừng ở document root nếu không tìm thấy.

#### Multiple actions với delegation:

```javascript
// ✅ Xử lý nhiều loại action trong 1 listener
document.querySelector('.product-list').addEventListener('click', (event) => {
  const addToCartBtn = event.target.closest('[data-action="add-to-cart"]');
  const wishlistBtn = event.target.closest('[data-action="wishlist"]');
  const shareBtn = event.target.closest('[data-action="share"]');

  if (addToCartBtn) {
    const productId = addToCartBtn.closest('.product-card').dataset.id;
    addToCart(productId);
    return;
  }

  if (wishlistBtn) {
    toggleWishlist(wishlistBtn.dataset.productId);
    return;
  }

  if (shareBtn) {
    shareProduct(shareBtn.dataset.url);
    return;
  }
});
```

**Trade-offs của Event Delegation:**

```
Ưu điểm:
✅ Memory: 1 listener thay vì N listeners
✅ Dynamic elements: tự động work với elements thêm vào sau
✅ Simpler cleanup: remove 1 listener thay vì N
✅ Performance: ít GC pressure từ listener objects

Nhược điểm:
❌ closest() traverse có cost (nhỏ, nhưng có)
❌ Event phải bubble — không work với non-bubbling events (focus, blur)
❌ Logic phức tạp hơn khi có nhiều nested interactive elements
❌ stopPropagation trên child sẽ prevent delegation
```

### E — stopPropagation, stopImmediatePropagation, preventDefault

Ba methods này giải quyết vấn đề khác nhau và thường bị nhầm lẫn.

#### `stopPropagation()`

```javascript
// Dừng event di chuyển lên/xuống DOM tree
// Listeners trên elements khác KHÔNG fire
// Listeners khác trên CÙNG element VẪN fire

button.addEventListener('click', (e) => {
  e.stopPropagation(); // Dừng bubble — parent listeners không nhận được
  console.log('button listener 1');
});

button.addEventListener('click', (e) => {
  console.log('button listener 2'); // VẪN fire — cùng element
});

div.addEventListener('click', (e) => {
  console.log('div listener'); // KHÔNG fire — bị stopPropagation
});

// Click button → output: "button listener 1", "button listener 2"
```

#### `stopImmediatePropagation()`

```javascript
// Dừng propagation VÀ dừng các listeners khác trên cùng element

button.addEventListener('click', (e) => {
  e.stopImmediatePropagation();
  console.log('button listener 1');
});

button.addEventListener('click', (e) => {
  console.log('button listener 2'); // KHÔNG fire
});

div.addEventListener('click', (e) => {
  console.log('div listener'); // KHÔNG fire
});

// Click button → output: "button listener 1" only
```

#### `preventDefault()`

```javascript
// Ngăn browser thực hiện default action của event
// Propagation KHÔNG bị dừng — event vẫn bubble bình thường

// Ví dụ default actions:
// click trên <a href> → navigate
// click trên <input type="submit"> → submit form
// keydown Space/ArrowDown trên page → scroll
// contextmenu → show browser context menu
// wheel → scroll page
// touchmove → scroll trên mobile

// ❌ Anti-pattern phổ biến: dùng để prevent navigation
link.addEventListener('click', (e) => {
  e.preventDefault(); // Ngăn navigation
  e.stopPropagation(); // Thừa nếu chỉ cần prevent navigation
  doSomething();
});

// Check xem default đã bị prevented chưa:
element.addEventListener('click', (e) => {
  if (e.defaultPrevented) {
    // Listener khác đã gọi preventDefault
    return;
  }
  // ...
});
```

#### `event.cancelable` — không phải event nào cũng cancelable

```javascript
// Chỉ cancelable events mới respond với preventDefault
document.addEventListener('scroll', (e) => {
  console.log(e.cancelable); // false — scroll event không cancelable!
  e.preventDefault(); // Không có tác dụng
});

// scroll KHÔNG cancelable vì: khi scroll event fire,
// scroll đã xảy ra rồi (không thể undo)
// Muốn prevent scroll phải dùng CSS: overflow: hidden
// hoặc preventDefault trên touchmove/wheel (trước khi scroll)

document.addEventListener(
  'wheel',
  (e) => {
    console.log(e.cancelable); // true (nếu passive: false)
    e.preventDefault(); // Prevent scroll — work!
  },
  { passive: false },
);
```

### F — Custom Events

#### Tạo và dispatch

```javascript
// Cách 1: Event (không có custom data)
const event = new Event('my-event', {
  bubbles: true, // default: false
  cancelable: true, // default: false
  composed: true, // Có vượt qua Shadow DOM boundary không — default: false
});

// Cách 2: CustomEvent (có custom data)
const event = new CustomEvent('product:added', {
  bubbles: true,
  cancelable: true,
  detail: {
    // detail: payload của event
    productId: '123',
    quantity: 2,
    price: 99.99,
  },
});

element.dispatchEvent(event);
```

#### `composed: true` — vượt qua Shadow DOM

```javascript
// Shadow DOM tạo encapsulation boundary
// Events mặc định không vượt qua boundary này

class MyButton extends HTMLElement {
  connectedCallback() {
    this.attachShadow({ mode: 'open' });
    this.shadowRoot.innerHTML = '<button>Click me</button>';

    this.shadowRoot.querySelector('button').addEventListener('click', () => {
      // Event này sẽ vượt qua Shadow DOM boundary:
      this.dispatchEvent(
        new CustomEvent('button-clicked', {
          bubbles: true,
          composed: true, // QUAN TRỌNG: không có cái này, event dừng ở shadow boundary
          detail: { timestamp: Date.now() },
        }),
      );
    });
  }
}
```

#### Event-driven Architecture Pattern

```javascript
// Pattern: Loosely coupled components qua custom events
// Thay vì direct method calls, components communicate qua events

class EventBus {
  #bus = document.createElement('div'); // Lightweight event target

  emit(event, detail = {}) {
    this.#bus.dispatchEvent(new CustomEvent(event, { detail, bubbles: false }));
  }

  on(event, handler, options = {}) {
    this.#bus.addEventListener(event, handler, options);
    // Return cleanup function
    return () => this.#bus.removeEventListener(event, handler);
  }

  once(event, handler) {
    return this.on(event, handler, { once: true });
  }
}

const bus = new EventBus();

// Cart component
const unsubscribe = bus.on('user:logout', () => {
  clearCart();
});

// Auth component
bus.emit('user:logout', { userId: '123' });

// Cleanup khi component unmount:
unsubscribe();
```

### G — Pointer Events vs Mouse Events vs Touch Events 🟡

#### Tại sao có 3 hệ thống event?

```
Lịch sử:
1990s: Mouse events — chỉ có chuột
2000s: Touch events — iPhone ra đời, Apple tạo Touch Events spec
2012:  Pointer Events — W3C unify chuột + touch + stylus thành 1 API
```

**Mouse Events:** Chỉ từ mouse/trackpad. Không fire trên touch screens.

**Touch Events:** Chỉ từ touch. Không fire khi dùng chuột (trừ emulation).

**Pointer Events:** Fire từ mọi pointing device — mouse, touch, stylus, pen. Recommended modern API.

#### Pointer Events — Unified API

```javascript
// Một event handler xử lý mọi input type:
element.addEventListener('pointerdown', (e) => {
  console.log(e.pointerType); // "mouse" | "touch" | "pen"
  console.log(e.pointerId); // Unique ID — track multiple simultaneous pointers
  console.log(e.pressure); // 0.0-1.0 (stylus pressure)
  console.log(e.width); // Contact area width (touch)
  console.log(e.height); // Contact area height (touch)
  console.log(e.isPrimary); // True cho primary pointer (first touch, hoặc mouse)
});
```

**Pointer capture — quan trọng cho drag:**

```javascript
// Vấn đề với drag: user drag nhanh, pointer ra ngoài element
// → pointermove không fire nữa → drag bị stuck

element.addEventListener('pointerdown', (e) => {
  // Capture pointer: tất cả pointer events từ pointerId này
  // sẽ được dispatch tới element này, dù pointer ở đâu
  element.setPointerCapture(e.pointerId);
});

element.addEventListener('pointermove', (e) => {
  // Vẫn fire kể cả khi pointer ra ngoài element
  updateDragPosition(e.clientX, e.clientY);
});

element.addEventListener('pointerup', (e) => {
  element.releasePointerCapture(e.pointerId);
  finalizeDrag();
});
```

#### Touch Events — Khi vẫn cần

```javascript
// Multi-touch gestures cần Touch Events vì Pointer Events
// không expose tất cả touch points trong 1 event object
document.addEventListener(
  'touchstart',
  (e) => {
    console.log(e.touches); // Tất cả current touches trên screen
    console.log(e.targetTouches); // Touches trên element cụ thể này
    console.log(e.changedTouches); // Touches thay đổi trong event này

    if (e.touches.length === 2) {
      // Pinch-to-zoom gesture
      const dx = e.touches[0].clientX - e.touches[1].clientX;
      const dy = e.touches[0].clientY - e.touches[1].clientY;
      initialDistance = Math.sqrt(dx * dx + dy * dy);
    }
  },
  { passive: true },
); // passive: true nếu không cancel scroll
```

### H — Keyboard Events 🟡

#### keydown, keyup, keypress (deprecated)

```javascript
document.addEventListener('keydown', (e) => {
  console.log(e.key); // "a", "Enter", "ArrowUp", "Shift", v.v.
  console.log(e.code); // "KeyA", "Enter", "ArrowUp", "ShiftLeft"
  console.log(e.keyCode); // deprecated — dùng e.key thay thế

  // Modifiers:
  console.log(e.ctrlKey); // Ctrl pressed
  console.log(e.shiftKey); // Shift pressed
  console.log(e.altKey); // Alt pressed
  console.log(e.metaKey); // Cmd (Mac) hoặc Win key

  // e.repeat: true nếu key đang được hold down
  console.log(e.repeat);
});
```

**`e.key` vs `e.code` — phân biệt quan trọng:**

```
e.key:  Giá trị của phím theo layout bàn phím hiện tại
        "a" hoặc "A" (tùy Shift), "é", "ñ"
        Thay đổi theo locale/layout!

e.code: Vị trí vật lý của phím trên bàn phím chuẩn US
        "KeyA", "KeyQ" (luôn là "KeyA" dù user dùng AZERTY layout)
        Không đổi theo locale!

Dùng e.key khi:    Quan tâm đến character (text input, search)
Dùng e.code khi:   Quan tâm đến vị trí phím (game controls: WASD)
```

```javascript
// Game controls — dùng e.code (vị trí phím, không phụ thuộc locale)
document.addEventListener('keydown', (e) => {
  switch (e.code) {
    case 'KeyW':
    case 'ArrowUp':
      moveUp();
      break;
    case 'KeyS':
    case 'ArrowDown':
      moveDown();
      break;
    case 'KeyA':
    case 'ArrowLeft':
      moveLeft();
      break;
    case 'KeyD':
    case 'ArrowRight':
      moveRight();
      break;
  }
});

// Keyboard shortcut — dùng e.key (character)
document.addEventListener('keydown', (e) => {
  if ((e.ctrlKey || e.metaKey) && e.key === 's') {
    e.preventDefault(); // Prevent browser save dialog
    saveDocument();
  }
});
```

#### IME Composition Events — quan trọng cho CJK

IME (Input Method Editor) là software keyboard cho tiếng Trung, Nhật, Hàn, Việt (Telex/VNI) — cho phép gõ nhiều phím để tạo ra một character.

```javascript
// Vấn đề: keydown fire trong khi user đang compose
// ví dụ: user gõ "ni" để ra "你" (tiếng Trung)
// keydown fire cho 'n' và 'i' — nhưng chưa phải final character

input.addEventListener('compositionstart', (e) => {
  // User bắt đầu IME composition
  isComposing = true;
});

input.addEventListener('compositionupdate', (e) => {
  // Intermediate composition string
  console.log(e.data); // "ni" — chưa hoàn thành
});

input.addEventListener('compositionend', (e) => {
  // Composition hoàn thành
  isComposing = false;
  console.log(e.data); // "你" — character cuối cùng
  processInput(e.data);
});

// Xử lý input đúng cách với IME:
input.addEventListener('keydown', (e) => {
  if (e.isComposing || e.keyCode === 229) {
    // e.isComposing: true khi đang trong IME session
    // keyCode 229: magic number cho IME events
    return; // Bỏ qua — chờ compositionend
  }

  if (e.key === 'Enter') {
    submitSearch(input.value);
  }
});
```

### I — Input vs Change Events 🟢

```javascript
// input event: fire SAU MỖI thay đổi (mỗi keystroke, paste, v.v.)
// change event: fire khi element MẤT FOCUS và giá trị đã thay đổi
//               (hoặc ngay lập tức cho checkbox/radio/select)

const textInput = document.querySelector('input[type="text"]');

textInput.addEventListener('input', (e) => {
  // Fire: sau mỗi keystroke, paste, cut, drag-drop text
  // e.target.value: giá trị HIỆN TẠI
  // Dùng cho: real-time search, character counter, validation while typing
  searchSuggestions(e.target.value);
});

textInput.addEventListener('change', (e) => {
```

### Drag and Drop với Pointer Events

```javascript
class DraggableItem {
  #element;
  #isDragging = false;
  #startX = 0;
  #startY = 0;
  #currentX = 0;
  #currentY = 0;
  #controller = new AbortController(); // Cleanup tất cả listeners cùng lúc

  constructor(element) {
    this.#element = element;
    this.#setupListeners();
  }

  #setupListeners() {
    const { signal } = this.#controller;

    this.#element.addEventListener(
      'pointerdown',
      this.#handlePointerDown.bind(this),
      { signal },
    );

    // pointermove và pointerup trên document
    // để handle khi pointer ra ngoài element
    document.addEventListener(
      'pointermove',
      this.#handlePointerMove.bind(this),
      { signal, passive: true }, // passive: không cancel scroll
    );

    document.addEventListener('pointerup', this.#handlePointerUp.bind(this), {
      signal,
    });

    document.addEventListener(
      'pointercancel', // Fire khi browser cancel pointer (call, notification)
      this.#handlePointerUp.bind(this),
      { signal },
    );
  }

  #handlePointerDown(e) {
    if (e.button !== 0 && e.pointerType === 'mouse') return; // Chỉ left click

    this.#isDragging = true;
    this.#startX = e.clientX - this.#currentX;
    this.#startY = e.clientY - this.#currentY;

    // Capture pointer: pointermove sẽ fire kể cả khi ra ngoài element
    this.#element.setPointerCapture(e.pointerId);

    this.#element.style.cursor = 'grabbing';
    this.#element.style.userSelect = 'none'; // Prevent text selection during drag

    // Dùng will-change để tạo compositing layer
    // → drag animation chỉ ảnh hưởng composite step, không relayout
    this.#element.style.willChange = 'transform';
  }

  #handlePointerMove(e) {
    if (!this.#isDragging) return;

    this.#currentX = e.clientX - this.#startX;
    this.#currentY = e.clientY - this.#startY;

    // transform thay vì top/left — chỉ trigger composite, không layout
    this.#element.style.transform = `translate(${this.#currentX}px, ${this.#currentY}px)`;
  }

  #handlePointerUp() {
    if (!this.#isDragging) return;

    this.#isDragging = false;
    this.#element.style.cursor = '';
    this.#element.style.userSelect = '';
    this.#element.style.willChange = 'auto'; // Release compositing layer

    // Dispatch custom event với final position
    this.#element.dispatchEvent(
      new CustomEvent('item:dropped', {
        bubbles: true,
        detail: { x: this.#currentX, y: this.#currentY },
      }),
    );
  }

  destroy() {
    // Remove tất cả listeners cùng lúc
    this.#controller.abort();
  }
}
```

```javascript
// Accessible dropdown menu với keyboard navigation
class AccessibleDropdown {
  #trigger;
  #menu;
  #items;
  #isOpen = false;
  #activeIndex = -1;
  #controller = new AbortController();

  constructor(container) {
    this.#trigger = container.querySelector('[data-trigger]');
    this.#menu = container.querySelector('[data-menu]');
    this.#items = Array.from(this.#menu.querySelectorAll('[role="menuitem"]'));
    this.#setup();
  }

  #setup() {
    const { signal } = this.#controller;

    // Click trigger để toggle
    this.#trigger.addEventListener('click', () => this.#toggle(), { signal });

    // Keyboard trên trigger
    this.#trigger.addEventListener(
      'keydown',
      (e) => {
        // e.key không phụ thuộc layout — đúng cho navigation keys
        switch (e.key) {
          case 'Enter':
          case ' ': // Space
            e.preventDefault();
            this.#toggle();
            break;
          case 'ArrowDown':
            e.preventDefault();
            this.#open();
            this.#focusItem(0);
            break;
          case 'Escape':
            this.#close();
            break;
        }
      },
      { signal },
    );

    // Keyboard trong menu
    this.#menu.addEventListener(
      'keydown',
      (e) => {
        switch (e.key) {
          case 'ArrowDown':
            e.preventDefault(); // Prevent page scroll
            this.#focusItem(
              Math.min(this.#activeIndex + 1, this.#items.length - 1),
            );
            break;
          case 'ArrowUp':
            e.preventDefault();
            if (this.#activeIndex <= 0) {
              this.#trigger.focus(); // Return to trigger
              this.#activeIndex = -1;
            } else {
              this.#focusItem(this.#activeIndex - 1);
            }
            break;
          case 'Home':
            e.preventDefault();
            this.#focusItem(0);
            break;
          case 'End':
            e.preventDefault();
            this.#focusItem(this.#items.length - 1);
            break;
          case 'Escape':
            this.#close();
            this.#trigger.focus();
            break;
          case 'Tab':
            // Allow natural tab — close menu
            this.#close();
            break;
        }
      },
      { signal },
    );

    // Event delegation cho menu items
    this.#menu.addEventListener(
      'click',
      (e) => {
        const item = e.target.closest('[role="menuitem"]');
        if (item) {
          this.#selectItem(item);
        }
      },
      { signal },
    );

    // Close khi click outside (capture phase — trước bất kỳ bubble listener nào)
    document.addEventListener(
      'click',
      (e) => {
        if (
          this.#isOpen &&
          !this.#trigger.contains(e.target) &&
          !this.#menu.contains(e.target)
        ) {
          this.#close();
        }
      },
      { capture: true, signal },
    );
  }

  #focusItem(index) {
    this.#activeIndex = index;
    this.#items[index]?.focus();
  }

  #toggle() {
    this.#isOpen ? this.#close() : this.#open();
  }

  #open() {
    this.#isOpen = true;
    this.#menu.hidden = false;
    this.#trigger.setAttribute('aria-expanded', 'true');
  }

  #close() {
    this.#isOpen = false;
    this.#menu.hidden = true;
    this.#trigger.setAttribute('aria-expanded', 'false');
    this.#activeIndex = -1;
  }

  #selectItem(item) {
    this.#trigger.dispatchEvent(
      new CustomEvent('dropdown:select', {
        bubbles: true,
        detail: { value: item.dataset.value, label: item.textContent.trim() },
      }),
    );
    this.#close();
    this.#trigger.focus();
  }

  destroy() {
    this.#controller.abort();
  }
}
```

## CÂU HỎI ÔN TẬP — (Dành cho NotebookLM)

### Câu 1

**Hỏi:** Giải thích tại sao `passive: true` cải thiện scroll performance. Browser làm gì khác nhau khi listener là passive vs non-passive?

**Đáp án chi tiết:** Scroll trên mobile và wheel scroll trên desktop có thể được xử lý hoàn toàn trên compositor thread — GPU thread tách biệt với main thread JavaScript. Compositor thread biết scroll offset và layer positions, có thể update visual mà không cần main thread. Vấn đề xảy ra khi có non-passive `touchstart`, `touchmove`, hoặc `wheel` event listeners: browser không biết trước listener có gọi `preventDefault()` để cancel scroll không. Nếu listener gọi `preventDefault()`, scroll phải bị cancel hoàn toàn. Vì không biết trước, browser PHẢI chờ main thread execute listener trước khi bắt đầu scroll. Nếu main thread đang bận (JavaScript execution, layout, paint), scroll bị delay — user thấy jank.

Với `passive: true`, developer cam kết với browser rằng `preventDefault()` sẽ không được gọi trong listener này. Browser nhận cam kết này và có thể bắt đầu scroll ngay lập tức trên compositor thread, đồng thời notify main thread để execute listener async. Listener vẫn execute, nhưng không block scroll. Từ Chrome 56+, browser tự set `passive: true` cho `touchstart` và `touchmove` trên window/document để tránh jank mặc định — đây là lý do gọi `preventDefault()` trong những listeners này tạo ra warning console.

### Câu 2

**Hỏi:** Điều gì xảy ra chính xác khi bạn gọi `stopPropagation()` trong capture phase? Event delegation có bị ảnh hưởng không?

**Đáp án chi tiết:** `stopPropagation()` trong capture phase dừng event tại element đó — event không tiếp tục đi xuống các descendant elements, không đến target, và không bubble lên. Tất cả listeners trên ancestor elements phía dưới (trong capture phase) và trên target và tất cả ancestors trong bubble phase đều không nhận event.

Event delegation thường đặt listener trên parent/container với bubble phase. Nếu `stopPropagation()` được gọi tại một child element trong capture phase, event không đến target và không bubble — delegate listener trên container sẽ không fire. Đây là bug phổ biến khi mix third-party libraries (stop propagation) với event delegation patterns.

Ngược lại, nếu `stopPropagation()` được gọi tại target trong bubble phase: nếu delegate listener ở ancestor sử dụng bubble phase, nó không nhận event. Nhưng nếu delegate listener sử dụng capture phase (`{ capture: true }`), nó vẫn nhận event vì capture phase xảy ra trước bubble. Đây là một trong số ít trường hợp hợp lý để dùng capture phase — intercept events trước khi child elements có cơ hội stop propagation.

### Câu 3

**Hỏi:** Đoạn code sau có vấn đề gì? Phân tích theo cả correctness và performance.

```javascript
const items = document.getElementsByClassName('list-item');

for (let i = 0; i < items.length; i++) {
  items[i].addEventListener('click', function (e) {
    items[i].classList.add('selected');
    e.stopPropagation();
  });
}
```

**Đáp án chi tiết:** Có nhiều vấn đề:

**Correctness — Live collection + closure:**`getElementsByClassName` trả về live HTMLCollection. Biến `i` trong closure của event handler capture variable `i` từ for loop, nhưng `let` tạo block scope mới cho mỗi iteration nên `i` được capture đúng — đây không phải bug với `let`. Tuy nhiên, `items` là live collection — nếu trong handler gọi `items[i].classList.add('selected')` và sau đó elements được add/remove, `items[i]` có thể trỏ sai element. Ngoài ra, dùng `items[i]` bên trong handler (sau khi click xảy ra) thay vì `e.currentTarget` là fragile — nếu DOM thay đổi giữa lúc listener attach và lúc handler fire, `i` có thể không còn đúng.

**Performance — N listeners thay vì delegation:** Với 1000 items, có 1000 event listener objects trong memory. Mỗi khi thêm items mới vào DOM, phải manually attach listeners. `stopPropagation()` trong mỗi listener ngăn event delegation patterns của code khác hoạt động.

**Fix:**

```javascript
// Event delegation + e.currentTarget
document.querySelector('.list').addEventListener('click', (e) => {
  const item = e.target.closest('.list-item');
  if (!item) return;

  // Deselect previous
  document.querySelector('.list-item.selected')?.classList.remove('selected');

  item.classList.add('selected');
  // stopPropagation không cần nếu không có specific reason
});
```
