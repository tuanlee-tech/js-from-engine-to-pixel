### A — requestIdleCallback: Tasks trong thời gian rảnh 🟡

#### A.1 — Vấn đề: Main thread luôn bận

Browser phải hoàn thành mỗi frame trong 16.67ms (60fps). Pipeline trong mỗi frame:

```
Input → JS (rAF) → Style → Layout → Paint → Composite → Display
│─────────────────────────────────────────────────────────────│
0ms                                                        16.67ms
```

Nếu pipeline xong trước 16.67ms, còn **idle time** — thời gian main thread không làm gì. `requestIdleCallback` (rIC) cho phép chạy code trong khoảng thời gian này mà **không ảnh hưởng frame timing**.

```
Frame với idle time:
│ Input │ rAF │ Style │ Layout │ Paint │ Composite │ IDLE  │ │
0ms                                                  12ms  16.67ms
                                                      ↑
                                              rIC chạy ở đây
```

#### A.2 — Cơ chế của rIC

```javascript
const idleCallbackId = requestIdleCallback(
  (deadline) => {
    // deadline.timeRemaining(): ms còn lại trong idle period
    // deadline.didTimeout: true nếu timeout option đã hết

    // Chạy tasks nhỏ cho đến khi gần hết idle time
    while (deadline.timeRemaining() > 1 && taskQueue.length > 0) {
      // timeRemaining() > 1: giữ lại 1ms buffer để browser react
      const task = taskQueue.shift();
      task();
    }

    // Nếu còn tasks: schedule tiếp
    if (taskQueue.length > 0) {
      requestIdleCallback(processQueue);
    }
  },
  {
    timeout: 2000, // Nếu sau 2000ms vẫn chưa có idle time
    // → force execute dù không idle
    // → deadline.didTimeout === true
  },
);

// Cancel nếu cần:
cancelIdleCallback(idleCallbackId);
```

**`deadline.timeRemaining()` trả về 0 khi:**

- Frame mới sắp bắt đầu (cần giữ 16.67ms budget)
- User input xảy ra (browser cần respond ngay)
- Một rIC task khác đang chờ với `didTimeout: true`

#### A.3 — Idle Period Types

```
Type 1: Inter-frame idle
  Xảy ra khi frame xong trước 16.67ms
  Duration: vài ms, không predictable
  Ví dụ: frame chỉ tốn 10ms → 6ms idle

Type 2: Long idle period
  Xảy ra khi không có animation/interaction
  User không scroll, không click, không animate
  Duration: có thể hàng trăm ms
  Browser cap tối đa 50ms để không block input response
  (Nếu user click trong lúc rIC đang chạy → dừng, handle input trước)
```

#### A.4 — Khi nào dùng rIC (và khi nào không)

```javascript
// ✅ ĐÚNG: Tasks không time-sensitive, không modify DOM
// Phân tích data
requestIdleCallback(() => {
  const stats = analyzeUserBehavior(events); // pure computation
  sendToAnalytics(stats); // async, không block UI
});

// ✅ ĐÚNG: Pre-compute data cho future use
requestIdleCallback(() => {
  // Pre-warm cache khi user idle
  prefetchProductRecommendations(userId);
});

// ✅ ĐÚNG: Cleanup tasks
requestIdleCallback(() => {
  clearExpiredCacheEntries();
  pruneEventLog();
});

// ❌ SAI: DOM mutations trong rIC
requestIdleCallback(() => {
  element.style.color = 'red'; // ❌ Có thể trigger layout/paint
  // rIC không đảm bảo không trigger extra frame
});

// ❌ SAI: Tasks cần kết quả ngay
requestIdleCallback(() => {
  // rIC không có timing guarantee (chỉ có khi idle)
  // Nếu user busy → có thể không chạy trong vài giây
  updateCriticalUI(); // ❌ Dùng rAF hoặc microtask
});

// ❌ SAI: Animation logic trong rIC
requestIdleCallback(() => {
  // rIC timing không sync với frame boundary
  animateElement(); // ❌ Dùng rAF
});
```

#### A.5 — Task Scheduler Pattern — production grade

```javascript
class IdleTaskScheduler {
  #queue = [];
  #isScheduled = false;
  #callbackId = null;

  // Schedule một task với optional priority
  schedule(task, { priority = 'idle', timeout } = {}) {
    if (priority === 'immediate') {
      // High priority: microtask
      queueMicrotask(task);
      return;
    }

    if (priority === 'next-frame') {
      // Medium priority: next rAF
      requestAnimationFrame(task);
      return;
    }

    // Low priority: idle time
    this.#queue.push({ task, timeout });
    this.#maybeSchedule();
  }

  #maybeSchedule() {
    if (this.#isScheduled) return;
    this.#isScheduled = true;

    // Chọn timeout từ task có timeout nhỏ nhất (most urgent idle task)
    const minTimeout = this.#queue.reduce(
      (min, { timeout }) => (timeout ? Math.min(min, timeout) : min),
      Infinity,
    );

    this.#callbackId = requestIdleCallback(
      this.#flush.bind(this),
      minTimeout < Infinity ? { timeout: minTimeout } : undefined,
    );
  }

  #flush(deadline) {
    this.#isScheduled = false;

    while (this.#queue.length > 0) {
      // Kiểm tra có đủ thời gian không
      if (!deadline.didTimeout && deadline.timeRemaining() < 1) {
        break; // Dừng, chờ idle tiếp theo
      }

      const { task } = this.#queue.shift();
      try {
        task();
      } catch (err) {
        console.error('Idle task error:', err);
      }
    }

    // Còn tasks → schedule tiếp
    if (this.#queue.length > 0) {
      this.#maybeSchedule();
    }
  }

  cancel() {
    if (this.#callbackId) {
      cancelIdleCallback(this.#callbackId);
      this.#callbackId = null;
      this.#isScheduled = false;
    }
  }
}

// Sử dụng:
const scheduler = new IdleTaskScheduler();

// Analytics (không urgent, chạy khi idle)
scheduler.schedule(
  () => {
    sendAnalytics({ page: 'home', timeOnPage: 30 });
  },
  { priority: 'idle', timeout: 5000 },
); // Nhưng không muộn hơn 5s

// Pre-fetch (không urgent)
scheduler.schedule(
  () => {
    prefetchNextPage('/checkout');
  },
  { priority: 'idle' },
);

// Critical update (urgent)
scheduler.schedule(
  () => {
    updateCartCount(3);
  },
  { priority: 'next-frame' },
);
```

### B — OffscreenCanvas: Heavy rendering trong Worker 🟢

#### B.1 — Vấn đề: Canvas rendering block main thread

```javascript
// Canvas work nặng trên main thread:
const canvas = document.querySelector('canvas');
const ctx = canvas.getContext('2d');

function renderComplexChart(data) {
  // Vẽ 10,000 data points với gradients, shadows...
  // Tốn 50-100ms → block main thread → frame drops
  data.forEach((point) => {
    ctx.beginPath();
    ctx.arc(point.x, point.y, point.r, 0, Math.PI * 2);
    ctx.fill();
  });
}
```

#### B.2 — OffscreenCanvas: Move rendering sang Worker

```javascript
// Main thread:
const canvas = document.querySelector('canvas');

// Transfer canvas control sang Worker
// Sau transfer, main thread KHÔNG thể dùng canvas này nữa
const offscreen = canvas.transferControlToOffscreen();

const worker = new Worker('/chart-worker.js');
worker.postMessage(
  { type: 'init', canvas: offscreen },
  [offscreen], // Transfer object — không copy, transfer ownership
);

// Gửi data để render:
worker.postMessage({
  type: 'render',
  data: chartData,
});

// Worker thread (chart-worker.js):
let ctx;

self.addEventListener('message', (event) => {
  const { type, canvas, data } = event.data;

  if (type === 'init') {
    // Worker nhận OffscreenCanvas
    ctx = canvas.getContext('2d');
    return;
  }

  if (type === 'render') {
    // Rendering xảy ra HOÀN TOÀN trong Worker thread
    // Main thread không bị block
    renderChart(ctx, data);
    return;
  }
});

function renderChart(ctx, data) {
  // Heavy rendering — 100ms+ OK vì chạy trong Worker
  ctx.clearRect(0, 0, ctx.canvas.width, ctx.canvas.height);

  data.forEach((point) => {
    ctx.beginPath();
    ctx.arc(point.x, point.y, point.radius, 0, Math.PI * 2);
    ctx.fillStyle = point.color;
    ctx.fill();
  });
  // Browser tự composite kết quả lên page
}
```

**Cơ chế:** OffscreenCanvas có rendering context riêng trong Worker. Browser composite canvas output vào page frame mà không cần main thread. Main thread hoàn toàn free trong khi Worker đang vẽ.

**Limitations của OffscreenCanvas:**

```javascript
// KHÔNG available trong Worker:
document.querySelector(...)  // No DOM access
window.devicePixelRatio      // Không access — phải pass từ main thread
CSS animations               // Không có
requestAnimationFrame        // Không có — dùng setTimeout hoặc message-based loop

// AVAILABLE trong Worker:
fetch()                      // Network requests OK
WebGL context                // OffscreenCanvas hỗ trợ WebGL
ImageBitmap                  // Efficient image transfer
createImageBitmap()          // Decode images off main thread
```

### C — getComputedStyle: Cơ chế thật 🔴

#### C.1 — Inline style vs Computed style

```javascript
// el.style: chỉ trả về INLINE styles (set trực tiếp trên element)
el.style.color; // "" nếu color được set bởi CSS class
el.style.width; // "" nếu width từ stylesheet

// getComputedStyle: trả về RESOLVED style — giá trị cuối cùng
// sau khi apply cascade, inheritance, và resolve relative values
const computed = window.getComputedStyle(el);
computed.color; // "rgb(255, 0, 0)" — absolute value, không "red"
computed.width; // "320px" — absolute px, không "50%" hay "auto"
computed.fontSize; // "16px" — absolute, không "1rem"
```

#### C.2 — Cơ chế: Trigger Reflow

`getComputedStyle()` **trigger synchronous style recalculation** nếu styles dirty, và **có thể trigger layout** cho layout-dependent properties:

```javascript
// Properties cần layout để compute (trigger reflow):
computed.width; // "auto" → phải layout để biết actual px
computed.height; // "auto" → phải layout
computed.top; // % → phải layout

// Properties KHÔNG cần layout:
computed.color; // Pure style, không phụ thuộc layout
computed.backgroundColor;
computed.fontFamily;

// Trick: getComputedStyle trả về CSSStyleDeclaration (live object)
// Đọc property tại thời điểm access, không phải lúc gọi getComputedStyle
const style = window.getComputedStyle(el); // Chưa compute gì
el.classList.add('big'); // Thay đổi style
style.width; // Compute LÚC NÀY — reflect thay đổi mới nhất
```

#### C.3 — Pseudo-elements

```javascript
// getComputedStyle có thể đọc pseudo-element styles
const beforeStyle = window.getComputedStyle(el, '::before');
const afterStyle = window.getComputedStyle(el, '::after');

beforeStyle.content; // '"→"' (với quotes)
beforeStyle.width; // Kích thước của pseudo-element
beforeStyle.display; // "block", "inline", v.v.

// Use case: kiểm tra icon từ CSS content property
function getIconContent(element) {
  return window
    .getComputedStyle(element, '::before')
    .content.replace(/['"]/g, ''); // Remove quotes
}
```

### D — getBoundingClientRect và offset/client/scroll 🔴

#### D.1 — Bộ ba offsetWidth/clientWidth/scrollWidth

Đây là một trong những nguồn nhầm lẫn phổ biến nhất:

```
┌─────────────────────────────────────────────────────────────┐
│                    margin (không tính)                      │
│  ┌───────────────────────────────────────────────────────┐  │
│  │              border (tính trong offset)               │  │
│  │  ┌─────────────────────────────────────────────────┐  │  │
│  │  │             padding                             │  │  │
│  │  │  ┌───────────────────────────────────────────┐  │  │  │
│  │  │  │                                           │  │  │  │
│  │  │  │         content area                      │  │  │  │
│  │  │  │                                           │  │  │  │
│  │  │  │   (có thể overflow ra ngoài nếu overflow  │  │  │  │
│  │  │  │    không phải hidden)                     │  │  │  │
│  │  │  └───────────────────────────────────────────┘  │  │  │
│  │  │                                                  │  │  │
│  │  └─────────────────────────────────────────────────┘  │  │
│  └───────────────────────────────────────────────────────┘  │
└─────────────────────────────────────────────────────────────┘

offsetWidth  = border + padding + content (scrollbar nếu có)
clientWidth  = padding + content (không border, không scrollbar)
scrollWidth  = padding + content KỂ CẢ overflow (phần bị ẩn bởi scroll)
```

```javascript
// Ví dụ cụ thể:
// <div style="width:200px; padding:10px; border:5px solid; overflow:auto">
//   <div style="width:400px">long content</div>
// </div>

div.offsetWidth; // 230px (200 + 10*2 padding + 5*2 border)
div.clientWidth; // 220px (200 + 10*2 padding, không border)
// Nếu có scrollbar (~15px): 205px
div.scrollWidth; // 420px (400px content + 10*2 padding — full content width)

// scrollLeft: current scroll position
div.scrollLeft; // 0 (chưa scroll)
div.scrollLeft = 100; // Scroll 100px
div.scrollLeft; // 100
```

**Height variants:**

```javascript
div.offsetHeight; // Tổng height bao gồm border, padding, scrollbar
div.clientHeight; // Visible height: padding + content, không border, không scrollbar
div.scrollHeight; // Full height bao gồm cả overflow content không visible
div.scrollTop; // Current scroll position (vertical)

// Detect khi đã scroll đến bottom:
const isAtBottom =
  Math.abs(el.scrollHeight - el.scrollTop - el.clientHeight) < 1; // < 1 vì floating point
```

**offsetTop/offsetLeft — relative to offsetParent:**

```javascript
// offsetTop: khoảng cách từ top của element đến top của offsetParent
// offsetParent: nearest ancestor với position không phải static

// Get position relative to document (không phải offsetParent):
function getDocumentOffset(element) {
  let top = 0,
    left = 0;
  let el = element;
  while (el) {
    top += el.offsetTop;
    left += el.offsetLeft;
    el = el.offsetParent; // traverse up qua offsetParent chain
  }
  return { top, left };
}
// Nhưng đây trigger nhiều reflows — dùng getBoundingClientRect thay thế
```

#### D.2 — getBoundingClientRect: Viewport-relative

```javascript
const rect = element.getBoundingClientRect();
// DOMRect {
//   x: 100,      ← left edge, relative to viewport
//   y: 200,      ← top edge, relative to viewport
//   width: 300,
//   height: 150,
//   top: 200,    ← same as y
//   right: 400,  ← x + width
//   bottom: 350, ← y + height
//   left: 100,   ← same as x
// }

// Relative to VIEWPORT — không phải document!
// Khi user scroll xuống 500px:
// element ở top: 1000px từ document top
// getBoundingClientRect().top = 1000 - 500 = 500px

// Convert về document coordinates:
const rect = element.getBoundingClientRect();
const documentTop = rect.top + window.scrollY;
const documentLeft = rect.left + window.scrollX;

// getBoundingClientRect LUÔN trigger synchronous layout!
// (cần tính toán vị trí chính xác tại thời điểm gọi)
```

**Khi nào dùng getBoundingClientRect vs offset properties:**

```javascript
// getBoundingClientRect: Khi cần viewport-relative position
//   → Tooltip positioning
//   → Check nếu element visible trong viewport
//   → Drag and drop (so sánh với clientX/clientY của mouse)

// offsetWidth/Height: Khi chỉ cần size, không cần position
//   → Tính toán layout
//   → Set width của element khác dựa vào element này
//   → Kiểm tra element có bị overflow không

// Cả hai đều trigger reflow — cố gắng batch với nhau
// và tách khỏi write operations
```

### E — CSS Custom Properties (Variables) từ JS 🔴

#### E.1 — Cơ chế của CSS Custom Properties

CSS Custom Properties khác với regular CSS properties ở điểm quan trọng: chúng **không được interpreted bởi CSS engine** — chỉ được stored và substituted khi dùng với `var()`.

```css
:root {
  --primary-color: #3b82f6; /* Stored as string, không parse */
  --spacing-unit: 8; /* "8", không phải số */
  --transition: 300ms ease; /* String */
}

.button {
  /* var() substitutes giá trị khi apply */
  background: var(--primary-color);
  padding: calc(var(--spacing-unit) * 2px); /* Bug! "8" không phải "8px" */
  padding: calc(var(--spacing-unit) * 1px); /* Cũng bug */
}
```

**Đúng cách với units:**

```css
:root {
  --spacing-unit: 8px; /* Include unit trong value */
}
```

#### E.2 — Đọc và ghi Custom Properties từ JS

```javascript
// ĐỌC: getPropertyValue
const root = document.documentElement; // :root
const computed = window.getComputedStyle(root);

// Đọc từ :root
const primaryColor = computed.getPropertyValue('--primary-color');
// → " #3B82F6" (note: có thể có leading space — cần trim)
const trimmed = primaryColor.trim(); // "#3B82F6"

// Đọc từ element cụ thể (có thể override :root)
const buttonComputed = window.getComputedStyle(button);
const buttonColor = buttonComputed.getPropertyValue('--primary-color').trim();

// GHI: setProperty
root.style.setProperty('--primary-color', '#EF4444');
// Apply ngay lập tức — tất cả elements dùng var(--primary-color) update

// GHI với fallback-awareness:
root.style.setProperty('--spacing-unit', '12px');

// XÓA:
root.style.removeProperty('--primary-color');
// Sau khi xóa, inherit từ parent hoặc dùng fallback
```

#### E.3 — Theming Pattern — Production

```javascript
// Theme system dùng CSS Custom Properties
class ThemeManager {
  #root = document.documentElement;
  #storageKey = 'app-theme';

  #themes = {
    light: {
      '--color-bg': '#FFFFFF',
      '--color-bg-secondary': '#F9FAFB',
      '--color-text': '#111827',
      '--color-text-secondary': '#6B7280',
      '--color-primary': '#3B82F6',
      '--color-primary-hover': '#2563EB',
      '--color-border': '#E5E7EB',
      '--shadow-sm': '0 1px 2px rgba(0,0,0,0.05)',
      '--shadow-md': '0 4px 6px rgba(0,0,0,0.07)',
    },
    dark: {
      '--color-bg': '#111827',
      '--color-bg-secondary': '#1F2937',
      '--color-text': '#F9FAFB',
      '--color-text-secondary': '#9CA3AF',
      '--color-primary': '#60A5FA',
      '--color-primary-hover': '#93C5FD',
      '--color-border': '#374151',
      '--shadow-sm': '0 1px 2px rgba(0,0,0,0.3)',
      '--shadow-md': '0 4px 6px rgba(0,0,0,0.4)',
    },
  };

  constructor() {
    // Restore saved theme
    const saved = localStorage.getItem(this.#storageKey);
    const prefersDark = window.matchMedia(
      '(prefers-color-scheme: dark)',
    ).matches;
    this.apply(saved || (prefersDark ? 'dark' : 'light'));

    // Listen for system theme changes
    window
      .matchMedia('(prefers-color-scheme: dark)')
      .addEventListener('change', (e) => {
        if (!localStorage.getItem(this.#storageKey)) {
          this.apply(e.matches ? 'dark' : 'light');
        }
      });
  }

  apply(themeName) {
    const theme = this.#themes[themeName];
    if (!theme) return;

    // Batch apply tất cả variables cùng lúc — 1 style recalculation
    Object.entries(theme).forEach(([property, value]) => {
      this.#root.style.setProperty(property, value);
    });

    this.#root.dataset.theme = themeName;
    localStorage.setItem(this.#storageKey, themeName);

    // Dispatch event để components có thể react
    document.dispatchEvent(
      new CustomEvent('theme:changed', { detail: { theme: themeName } }),
    );
  }

  toggle() {
    const current = this.#root.dataset.theme;
    this.apply(current === 'dark' ? 'light' : 'dark');
  }

  // Dynamic custom theming (user picks color)
  setCustomColor(name, value) {
    // Validate hex color
    if (!/^#[0-9A-F]{6}$/i.test(value)) return;

    this.#root.style.setProperty(`--color-${name}`, value);
    // Generate accessible variants
    this.#root.style.setProperty(
      `--color-${name}-hover`,
      this.#darken(value, 10),
    );
  }

  #darken(hex, percent) {
    const num = parseInt(hex.replace('#', ''), 16);
    const r = Math.max(0, (num >> 16) - Math.round(2.55 * percent));
    const g = Math.max(0, ((num >> 8) & 0xff) - Math.round(2.55 * percent));
    const b = Math.max(0, (num & 0xff) - Math.round(2.55 * percent));
    return `#${((r << 16) | (g << 8) | b).toString(16).padStart(6, '0')}`;
  }
}

const theme = new ThemeManager();
```

#### E.4 — Animate với CSS Custom Properties

```javascript
// CSS Custom Properties KHÔNG natively animatable với CSS transitions
// Vì browser không biết type (color? length? number?)

// ❌ Không work:
// .el { transition: --opacity 300ms; } → Không có effect

// ✅ Workaround 1: @property (CSS Houdini) — register type
// @property --my-opacity {
//   syntax: '<number>';
//   initial-value: 1;
//   inherits: false;
// }
// Sau khi register → có thể transition!

// ✅ Workaround 2: Animate từ JS, update custom property
function animateCustomProperty(element, property, from, to, duration) {
  const startTime = performance.now();

  function frame(currentTime) {
    const elapsed = currentTime - startTime;
    const progress = Math.min(elapsed / duration, 1);

    // Easing function
    const eased = 1 - Math.pow(1 - progress, 3); // ease-out-cubic

    const current = from + (to - from) * eased;
    element.style.setProperty(property, current);

    if (progress < 1) {
      requestAnimationFrame(frame);
    }
  }

  requestAnimationFrame(frame);
}

// Animate opacity qua custom property:
animateCustomProperty(el, '--card-opacity', 0, 1, 300);
```

### F — CSS Houdini: Extend CSS Engine 🟡

Houdini là collection of APIs cho phép developer **hook vào CSS engine** — viết custom CSS properties, custom layouts, custom paint.

#### F.1 — @property: Register Custom Properties

```javascript
// JavaScript registration:
CSS.registerProperty({
  name: '--gradient-angle',
  syntax: '<angle>', // Type — browser biết cách animate!
  initialValue: '0deg',
  inherits: false,
});

// Hoặc CSS @property:
// @property --gradient-angle {
//   syntax: '<angle>';
//   initial-value: 0deg;
//   inherits: false;
```

### Scroll Position Tracker — Không thrashing

```javascript
// Track scroll progress cho reading indicator, parallax, v.v.
// Tránh getBoundingClientRect trong scroll handler

class ScrollTracker {
  #sections = [];
  #rafId = null;
  #ticking = false;
  #lastScrollY = 0;

  constructor(sectionSelector) {
    // Đo tất cả sections MỘT LẦN lúc init (và khi resize)
    this.#measureSections(sectionSelector);
    this.#setupListeners();
  }

  #measureSections(selector) {
    // Batch read — đọc tất cả positions cùng lúc
    // Chỉ 1 reflow cho toàn bộ
    this.#sections = Array.from(document.querySelectorAll(selector)).map(
      (el) => ({
        el,
        // Đọc document-relative position (không viewport-relative)
        // Để không phải đọc lại khi scroll
        top: el.offsetTop,
        height: el.offsetHeight,
        id: el.id,
      }),
    );
  }

  #setupListeners() {
    // passive: true — không block scroll
    window.addEventListener(
      'scroll',
      () => {
        this.#lastScrollY = window.scrollY;

        // rAF throttling — chỉ process 1 lần per frame dù scroll event fire nhiều
        if (!this.#ticking) {
          this.#ticking = true;
          requestAnimationFrame(() => {
            this.#update();
            this.#ticking = false;
          });
        }
      },
      { passive: true },
    );

    // Remeasure khi resize (debounced)
    let resizeTimeout;
    window.addEventListener(
      'resize',
      () => {
        clearTimeout(resizeTimeout);
        resizeTimeout = setTimeout(() => {
          this.#measureSections('[data-section]');
        }, 150);
      },
      { passive: true },
    );
  }

  #update() {
    const scrollY = this.#lastScrollY;
    const viewportHeight = window.innerHeight;

    this.#sections.forEach((section) => {
      // Tính toán từ stored values — không read DOM
      const relativeTop = section.top - scrollY;
      const progress = 1 - relativeTop / viewportHeight;
      const clampedProgress = Math.min(1, Math.max(0, progress));

      // Chỉ write — không read DOM trong update
      section.el.style.setProperty('--scroll-progress', clampedProgress);
    });

    // Update progress bar (chỉ write)
    const totalProgress =
      scrollY / (document.documentElement.scrollHeight - viewportHeight);
    document.documentElement.style.setProperty(
      '--page-progress',
      Math.min(1, totalProgress),
    );
  }
}
```

### getComputedStyle — Safe Reading Pattern

```javascript
// Utility để đọc computed styles hiệu quả
// Tránh triggering nhiều reflows

class StyleReader {
  // Cache computed style objects per element
  // (nhưng nhớ: computed style là live object — stale nếu styles change)
  #cache = new WeakMap();

  // Batch đọc nhiều properties từ cùng 1 element — 1 reflow
  getProperties(element, properties) {
    const computed = window.getComputedStyle(element);
    // Lấy tất cả properties trong 1 lần
    return properties.reduce((result, prop) => {
      result[prop] = computed.getPropertyValue(prop).trim();
      return result;
    }, {});
  }

  // Đọc CSS custom property với fallback
  getCustomProperty(element, property, fallback = '') {
    const computed = window.getComputedStyle(element);
    const value = computed.getPropertyValue(property).trim();
    return value || fallback;
  }

  // Parse numeric value với unit
  getNumericValue(element, property) {
    const computed = window.getComputedStyle(element);
    const raw = computed.getPropertyValue(property).trim();
    return {
      value: parseFloat(raw),
      unit: raw.replace(/[\d.+-]/g, ''),
    };
  }

  // Đọc nhiều elements cùng lúc — 1 batch
  batchRead(elements, properties) {
    // Tất cả reads xảy ra trước khi có bất kỳ write nào
    return elements.map((el) => ({
      element: el,
      styles: this.getProperties(el, properties),
      rect: el.getBoundingClientRect(), // Gộp luôn nếu cần
    }));
  }
}
```

```javascript
// Custom hook: schedule non-critical work với rIC
import { useEffect, useCallback, useRef } from 'react';

function useIdleCallback(callback, deps, options = {}) {
  const callbackRef = useRef(callback);
  callbackRef.current = callback; // Always latest

  useEffect(() => {
    if (!('requestIdleCallback' in window)) {
      // Fallback: setTimeout
      const id = setTimeout(() => callbackRef.current(), 1);
      return () => clearTimeout(id);
    }

    const id = requestIdleCallback(
      (deadline) => callbackRef.current(deadline),
      options,
    );

    return () => cancelIdleCallback(id);
  }, deps);
}
```

## CÂU HỎI ÔN TẬP — (Dành cho NotebookLM)

### Câu 1

**Hỏi:** `requestIdleCallback` và `requestAnimationFrame` khác nhau như thế nào về timing và use cases? Tại sao không nên modify DOM trong rIC callback?

**Đáp án chi tiết:** `requestAnimationFrame` callback chạy **một lần mỗi frame, ngay trước browser paint** — timing được đảm bảo sync với display refresh rate (60fps = mỗi 16.67ms). rAF thích hợp cho mọi thứ visual: animation, DOM updates cần reflect ngay, measurements cần chính xác với frame hiện tại.

`requestIdleCallback` chạy trong **idle time** — khoảng thời gian sau khi browser hoàn thành tất cả urgent work (input, rAF, style, layout, paint) và còn thời gian trước frame tiếp theo. Duration không được đảm bảo và không predictable — có thể vài ms hoặc không chạy trong vài giây nếu browser liên tục bận. Browser cap idle period tối đa 50ms để vẫn có thể respond với user input.

Không nên modify DOM trong rIC vì: DOM mutations có thể trigger style recalculation và layout. Nếu rIC modify DOM, browser sẽ phải recalculate styles và reflow — những operations này sẽ "borrow" time từ idle period và có thể push sang frame tiếp theo, làm cho frame đó dài hơn 16.67ms → jank. rIC được thiết kế cho pure computation: analytics processing, cache cleanup, prefetching data, pre-computing values — không phải visual updates.

### Câu 2

**Hỏi:** Giải thích sự khác biệt giữa `offsetWidth`, `clientWidth`, và `scrollWidth`. Viết code detect khi một element đang bị text overflow (content bị cắt bởi ellipsis).

**Đáp án chi tiết:** `offsetWidth` = border + padding + content width + scrollbar (nếu có). Là "visual size" của element bao gồm mọi thứ ngoại trừ margin. `clientWidth` = padding + content width, không bao gồm border và scrollbar. Là "usable inner space" — không gian thực sự available cho content. `scrollWidth` = padding + tổng width của content kể cả phần bị hidden bởi overflow. Nếu không có overflow, `scrollWidth === clientWidth`. Nếu content overflow, `scrollWidth > clientWidth`.

Detect text overflow (ellipsis):

```javascript
function isTextOverflowing(element) {
  // scrollWidth bao gồm text bị cắt
  // clientWidth chỉ bao gồm visible area
  // Nếu text overflow → scrollWidth > clientWidth
  return element.scrollWidth > element.clientWidth;
}

// Với inline elements, cần clone để đo natural width:
function isEllipsisActive(element) {
  // Method đáng tin cậy hơn: tạm thời remove overflow constraint

  clone.style.cssText = `
    position: fixed; visibility: hidden;
    overflow: visible; white-space: nowrap;
    width: auto; max-width: none;
  `;
  document.body.appendChild(clone);
  const isOverflowing = clone.offsetWidth > element.offsetWidth;
  document.body.removeChild(clone);
  return isOverflowing;
}
```

### Câu 3

**Hỏi:** CSS Custom Properties được lưu trữ và cascade như thế nào? Tại sao `element.style.setProperty('--color', 'red')` trên `:root` không override custom property đã được set trên `.card` bằng CSS?

**Đáp án chi tiết:** CSS Custom Properties follow **CSS cascade** như mọi property khác — specificity, scope, và inheritance đều áp dụng. Custom properties được **inherited by default** (trừ khi register với `inherits: false` qua `@property`).

Cascade resolution theo thứ tự: (1) Inline styles (`el.style.setProperty`), (2) Author stylesheets theo specificity/order, (3) User stylesheets, (4) Browser defaults. Khi `document.documentElement.style.setProperty('--color', 'red')` được gọi, nó set inline style trên `:root` element — high specificity nhưng chỉ áp dụng cho `:root` element.

Khi `.card` có `--color: blue` trong một CSS rule, đó là một **separate declaration scoped to `.card`**. CSS Custom Properties cascade per-element: mỗi element tìm giá trị của `--color` bằng cách đi từ element đó lên ancestors, lấy giá trị của "nearest ancestor với declaration" (hoặc chính element đó nếu có). `.card` có `--color: blue` declaration → khi `.card .title` dùng `var(--color)`, nó tìm thấy `blue` từ `.card` trước khi đến `:root`. `:root` có `--color: red` nhưng nó ở higher trong DOM tree → lower specificity trong cascade context của `.card` → bị override bởi `.card`'s declaration. Để thay đổi `--color` cho tất cả elements kể cả `.card`, phải set trực tiếp trên `.card`: `card.style.setProperty('--color', 'red')`.
